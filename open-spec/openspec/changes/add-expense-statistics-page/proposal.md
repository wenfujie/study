## Why

当前项目缺少一个用于录入和汇总花销的页面，用户无法直观看到各项支出及合计金额。现在补充该页面可以满足基础记账与快速核算的核心场景。

## What Changes

- 新增“花销统计页面”，提供可编辑表格，包含“花销项”和“花销金额”两列。
- 每一行两列均使用输入框，支持用户直接录入或修改内容。
- 金额列输入限制为数字，并约束最多两位小数。
- 页面在表格下方展示总金额 `amount`，并基于金额列输入实时动态计算。

## Capabilities

### New Capabilities
- `expense-statistics-page`: 提供花销项录入、金额格式约束（数字且两位小数）与实时总金额汇总展示。

### Modified Capabilities
- None.

## Impact

- Affected code: 前端页面与组件（预计在 `src/` 下新增或调整页面组件与状态计算逻辑）。
- APIs: 无新增后端 API 依赖，当前能力可在前端本地状态完成。
- Dependencies: 无强制新增依赖，优先使用现有框架能力实现输入校验与合计计算。
